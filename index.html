extendJS by chrisbod

A lightweight memory friendly javascript object extender (with requirejs in mind)

Having worked on some pretty massive HTML/Javascript frameworks, APIs and applications I'm pretty precious about memory. Javascript is seriously memory friendly when used the right way (use prototypical inheritance and as few closures as possible) - but requirejs uses one big closure and the bulk of objects being developed by people use closures liberally for private variable creation (often when unnecessary). So I wanted a way to extend my required objects (or any objects) which was:

  • memory friendly (reduce amount of closures)
  • prototypical inheritance friendly (allows types with almost no closures whatsoever)
  • capable of extending objects using many different filters so I could get just what I wanted whenever I wanted and probably a lot more besides
  • doesn't need to pollute the global scope
  • required friendly (PENDING...)
